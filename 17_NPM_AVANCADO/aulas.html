Apêndice A - avançando no NPM

Introdução

Comandos não muito comuns, para resolver problemas e aumentar produtividade.

Verificando a versão do NPM

npm [node] -v

Criar projetos rapidamente

npm init -y

Configuração de módulo

  1. criar a pasta do projeto
  2. antes de iniciar o projeto vamos na pasta e damos o comando npm config set chave-a-configurar valor
  3. ex 1.: npm config set init-author-name Paulo
  4. ex 2.: npm config set init-author-email paulo@teste.com
  5. ex 3.: npm config set init-licence MIT

quando criarmos um projeto com npm init -y nosso projeto irá com esses dados

Aqui podemos ver todas as configurações possíveis de serem feitas em um novo projeto npm

Exibindo configurações do npm

npm get

Deletando valores de configuração

npm config delete chave-a-deletar

Instalação de módulo global

Podemos ter acesso a módulos em qualquer projeto utilizando a instalação global. A instalação global permite que não instalemos diretamente no projeto módulos não necessários para a execução deste projeto, por exemplo, módulos de teste, que usamos no desenvolvimento, mas que não são necessários para que o usuário consiga usar corretamente o sistema.

npm i modulo-global -g

o pacote não aparece no package.json, mas é possível utilizá-lo, sendo assim, quando outro dev for rodar o projeto clonado, não haverá a instalação dos módulos globais utilizados, sendo que estes deverão ser linkados de maneira manual ou de outra forma por este dev

para rodar um programa que faz uso de um módulo global nós temos que primeiro dar o comando npm link modulo-global e depois sim rodar o projeto com o node programa.js

Remover módulos globais

npm remove modulo-global -g

Listando módulos instalados

npm list(ls) [--depth=n &|| -g]

depth indica a "profundidade" da dependência, começando em n=0, onde são listadas as dependências imediatas do projeto

-g são os módulos de dependência global, podemos associar com o depth

Limpando dependências não utilizadas

se apagarmos uma dependência escrita no package.json podemos dar o comando npm prune para o npm verificar quais dependências podem ser deletadas sem problemas

Procurando módulos pelo terminal

npm search palavra-chave

recebemos uma lista com informações que batem com a palavra-chave passada, seja no nome do módulo, sua descrição, autor e uma própria lista de palavras chave
