1. começou como cópia da pasta anterior
  2. em router foi criada a rota get /edit/:id com o callback como TaskController.updateTask
  3. updateTask é async
  4. o id vai ser recebido pelos parâmetros da url
  5. é executado o método findOne de Task como await. O retorno desse método é recebido pela constante task
  6. o método findOne recebe como parâmetros where e raw:true
  7. é renderizada a página task/edit que recebe a const task gerada pelo updateTask
  8. é criada a views/edit baseada em views/create, onde a action do form post é alterada para /tasks/edit
  9. é criada a rota post /edit de callback TaskController.updateTaskPost
  10. em TaskController foi criado o método async updateTaskPost que recebe o id, title e description da task pelo body da requisição. title e description são utilizados para a criação de um objeto task dentro do método
  11. é chamado o método update da classe Task, onde é passado como primeiro parâmetro o objeto task, com os dados a serem atualizados e um where procurando o id da tarefa
  12. ao final é redirecionado para /tasks
  13. view/all tem o ícone de editar alterado para conseguir acessar as rotas de edição
  14. é criada a rota post /updateStatus com o callback TaskController.toggleTaskStatus
  15. toggleTaskStatus é um método async para dar um update em um task específico no valor do atributo done para true ou false dependendo do valor que ele estava inicialmente
  16. em views/all o ícone de check vai ser agora um form post com action para /tasks/updateStatus
    • esse form tem dois input hidden (id e done) e um botão
  17. o professor fez algumas alterações em css e no views/all para gerar uma visualização de tarefas feitas
