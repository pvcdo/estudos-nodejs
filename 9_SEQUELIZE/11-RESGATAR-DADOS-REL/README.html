  2. em index.js nós vamos na rota get /users/edit/:id
    1. no método User.findOne nós removemos o atributo raw e incluímos o atributo "include: Address"
    2. dessa forma, o retorno de findOne também retornará a tabela address
    3. por ter alterado o raw, nós vamos receber um objeto grande e cheio de atributos e objetos além da tabela, portanto temos que, no res.render alterar o objeto que é passado para a página para --- user: user.get({plain:true}) ---
    4. esse método get vem do objeto retornado pelo findOne sem o raw, já o valor user (de user.get) imagino ser o nome da tabela no mysql
    5. toda essa parte serve para que não precisemos alterar as variáveis na página edituser.handlebars
  3. em edituser vamos colocar uma div com estilo que comportará um each para listar os endereços já cadastrados do usuário, essa div ficará abaixo do form de incluir endereço
  4. em models/Address, logo acima de Address.belongsTo, colocamos User.hasMany(Address). Isso serve para efetivar a ligação entre as tabelas
