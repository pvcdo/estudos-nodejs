1. começou como cópia da pasta anterior
  2. criou view/edituser.handlebars e baseou seu conteúdo com a addusers. Em edituser ele:
    • o action do form será /users/update com método post
    • ele colocou logo no começo do form principal um input hidden
    • nos inputs ele colocou o value como o {{user.__CAMPO__}}
    • criou um if para verificar se o usuário aceitou ou não newsletter
  3. em home ele colocou uma tag a que irá redirecionar para /users/edit/{{this.id}}
  4. em index ele criou a rota get /users/edit/:id, com callback sync que recebe o id passado na url, usa o método findOne de User como await e passa seu retorno para a const user, esse método com where e raw:true, no final renderiza a edituser, passando a const user para ela
  5. ainda em index criamos uma rota post /users/update que pega os parâmetros que vem no body da requisição, faz um tratamento condicional para mudar o valor de newsletter para true ou false, depois nós passamos dentro de um objeto os parâmetros vindos do body como argumento do método update do model User; ao final redirecionamos para a rota '/'
  6. o callback da rota /users/update é sync e o método update é await
  7. é possível perceber que no banco a data de update e creation são diferentes, indicando que o sequelize cuidou de atualizar esses campos
